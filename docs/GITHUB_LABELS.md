# GitHub Labels

Suggested labels for maintaining MuseDock Open.

## Type

- `bug`: Something is broken.
- `feature`: New user-facing capability.
- `docs`: Documentation changes.
- `provider`: Provider compatibility issue.
- `release`: Release packaging, build, or versioning.
- `privacy`: Local data, API key handling, or user privacy.

## Priority

- `priority: high`: Blocks normal use or release.
- `priority: medium`: Important but not blocking.
- `priority: low`: Nice to have.

## Status

- `needs reproduction`: Needs clear steps or a sample case.
- `needs design`: Needs product or UX decision.
- `blocked`: Waiting on external dependency or maintainer action.
- `good first issue`: Small, well-scoped contribution.

## Platform

- `macOS`: macOS-specific issue.
- `Windows`: Windows-specific issue.
- `Linux`: Linux-specific issue.

## Product Principles

Use labels to protect the product direction:

- Prefer simple defaults over exposing advanced AI parameters.
- Keep API keys out of exported files and logs.
- Make provider compatibility clear without making the main UI harder to use.
